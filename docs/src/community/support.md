# Support & Community

Not sure where to ask? Use this page to pick the right channel — community chat for quick questions, GitHub issues for tracked work, and the security policy for vulnerabilities.

## Choose the right channel

| I want to… | Use |
|------------|-----|
| Get quick help with integration or usage | [Developer Discord](https://discord.gg/egjX5JgQJ) |
| Report a bug or broken behavior | [Bug report](https://github.com/Opencomplai/opencomplai/issues/new?template=bug-report.yml) |
| Propose a feature or improvement | [Feature request](https://github.com/Opencomplai/opencomplai/issues/new?template=feature-request.yml) |
| Track a defined piece of work | [Task](https://github.com/Opencomplai/opencomplai/issues/new?template=task.yml) |
| Something else (no template fits) | [Blank issue](https://github.com/Opencomplai/opencomplai/issues/new?template=BLANK_ISSUE) |
| Report a **new** security vulnerability | [Security policy](https://github.com/Opencomplai/opencomplai/security/policy) — **do not** open a public issue |
| Track remediation of a known CVE | [Vulnerability triage](https://github.com/Opencomplai/opencomplai/issues/new?template=vuln-triage.md) |
| Discuss ideas or ask questions async | [GitHub Discussions](https://github.com/Opencomplai/opencomplai/discussions) |
| Raise a private concern | [open@opencomplai.com](mailto:open@opencomplai.com) |

## GitHub issue templates

| Template | When to use | Open |
|----------|-------------|------|
| Bug report | Something isn't working as documented or expected | [Create issue](https://github.com/Opencomplai/opencomplai/issues/new?template=bug-report.yml) |
| Feature request | Propose a new capability or improvement | [Create issue](https://github.com/Opencomplai/opencomplai/issues/new?template=feature-request.yml) |
| Task | A scoped piece of work with clear acceptance criteria | [Create issue](https://github.com/Opencomplai/opencomplai/issues/new?template=task.yml) |
| Blank issue | Your request doesn't fit the templates above | [Create issue](https://github.com/Opencomplai/opencomplai/issues/new?template=BLANK_ISSUE) |

Before opening a duplicate, search [existing issues](https://github.com/Opencomplai/opencomplai/issues).

## Security

!!! warning "Do not report new vulnerabilities in public issues"
    If you discover a **new** security vulnerability, follow our [security policy](https://github.com/Opencomplai/opencomplai/security/policy) and email **security@opencomplai.com**. Do not open a public GitHub issue.

The security policy covers scope, response timelines, and coordinated disclosure.

### Vulnerability triage (maintainers)

The [vulnerability triage template](https://github.com/Opencomplai/opencomplai/issues/new?template=vuln-triage.md) is for **tracking known** dependency or code vulnerabilities through remediation (e.g. after a private report or CI scan). It is not the channel for initial private vulnerability reports.
